# E-commerce website- Velvet Bloom

A fully responsive flower e-commerce website built from scratch with HTML and CSS — featuring a warm maroon, cream, and gold aesthetic designed to feel boutique and editorial rather than templated.

**Live Demo https://snehafincy09.github.io/ecommerce-page/** 

---

## About

Velvet Bloom is a concept flower shop showcasing three product categories — Bouquets, Plants, and Gifts — across 18 individual product cards, a video home section, and a fully responsive layout that adapts across desktop, tablet, and mobile breakpoints.

This project was built as a way to go deeper than basic layout practice: deliberate color theory, typography pairing, micro-interactions, accessibility considerations (focus states, semantic structure), and real responsive engineering across two breakpoints — all written by hand, no frameworks.

## Features

- **Custom design system** — maroon , cream , and gold color palette applied consistently across every section
- **18 product cards** across 3 categories (Bouquets, Plants, Gifts), each with image, name, description, price, and an "Add to Cart" action
- **Video homme section** with gradient overlay and smooth-scroll anchor navigation
- **Micro-interactions** — hover scale effects, color transitions, and shadow lifts on buttons and cards
- **Accessibility-conscious** — visible focus states on all interactive elements, semantic HTML structure
- **Fully responsive** — two custom breakpoints (tablet: 768px, mobile: 480px) restructuring grid columns, navigation, typography scale, and spacing for every screen size
- **Typography pairing** — Playfair Display for headings, Poppins for body text

## Tech Stack

- **HTML5** — semantic structure
- **CSS3** — Flexbox, CSS Grid, custom properties, media queries, transitions
- **Font Awesome** — icons
- **Google Fonts** — Playfair Display & Poppins
- **Pexels**  — Images

No frameworks, no libraries, no build tools — pure hand-written HTML/CSS to deepen fundamentals before moving into JavaScript and React.

## Responsive Design

| Breakpoint | Behavior |
|---|---|
| Desktop (default) | 3-column product grids, full-width nav |
| Tablet (≤768px) | 2-column grids, wrapping navigation |
| Mobile (≤480px) | Single-column grids, condensed typography, stacked nav |

## What I Learned

- Debugging real layout issues using browser DevTools
- Structuring CSS to avoid repetition
- Planning a mobile-first cascade across multiple breakpoints without relying on JavaScript
